Simulación

Prompts de edición generados para el proyecto

Esta página deja visible la simulación de los prompts nuevos para documentar los cambios pedidos en el trabajo.

Prompt 24

Acceso por roles

Reforzar la lógica de usuarios del proyecto sin usar JavaScript, Python, SQL ni ningún otro lenguaje. Crear una simulación visual de acceso por roles para que el administrador no pueda ser creado libremente desde la interfaz pública. Debe quedar explícito que el administrador requiere usuario y contraseña asignados por la plataforma, mientras que el propietario y el gestor free usan formularios de registro independientes. Mantener todo con HTML y CSS.

Prompt 25

12 puestos disponibles

Revisar la portada y el listado de propiedades del sitio PNK Inmobiliaria. Mostrar 12 puestos en total, considerando las 3 propiedades publicadas y 9 espacios reservados para futuras propiedades. No usar archivos externos ni scripts. Representar los espacios vacíos con HTML y CSS, manteniendo el diseño visual de una inmobiliaria profesional y la navegación entre módulos.

Prompt 26

Login y dashboard

Actualizar la página de login y el dashboard de administración para que el flujo de usuarios quede más cercano a los antecedentes del proyecto. El login debe mostrar los roles separados, incluir credenciales de demostración para el administrador y dejar claro que el acceso administrativo es restringido. El dashboard debe mostrar CRUD de usuarios y propiedades, además de indicar que el administrador habilita, da de baja y administra publicaciones. Usar solo HTML y CSS y conservar el estilo global del sitio.
