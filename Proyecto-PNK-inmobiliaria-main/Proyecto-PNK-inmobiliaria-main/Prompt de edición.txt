Proyecto: PNK Inmobiliaria
Restricción indicada: trabajar solo con HTML y CSS

Prompt 1:
Actúa como asistente de desarrollo frontend dentro de Visual Studio Code. Necesito mejorar el proyecto PNK Inmobiliaria sin borrar los avances existentes de mi compañera. El proyecto debe seguir siendo un sitio estático hecho solo con HTML y CSS. Mejora el diseño visual general, crea un archivo CSS global, define una paleta de colores profesional para una inmobiliaria, usa una tipografía moderna, mejora los espaciados, botones, formularios, tarjetas de propiedades y navegación. También considera accesibilidad, contraste, foco visible y diseño responsive.

Prompt 2:
Revisa todas las páginas HTML del proyecto PNK Inmobiliaria y conecta cada una con un archivo global css/styles.css. Mantén la navegación entre todos los módulos: inicio, registro propietario, registro gestor, login, recuperar contraseña, buscador, listado de propiedades, publicar propiedad, detalle de propiedad, mapa, dashboard, quiénes somos y contacto. Corrige textos con problemas de acentos o codificación y mejora la estructura semántica usando header, nav, main, section, article y footer.

Prompt 3:
Mejora la accesibilidad del sitio. Agrega textos alternativos descriptivos en las imágenes, labels asociados a todos los inputs, selects y textarea, aria-label cuando sea necesario, botón o enlace para saltar al contenido principal, estados de foco visibles en enlaces, botones y campos de formulario. Asegúrate de que el contraste de colores sea legible y que el sitio pueda navegarse correctamente con teclado.

Prompt 4:
Diseña la página principal index.html como portada de una inmobiliaria. Debe tener un encabezado visual, nombre PNK Inmobiliaria, menú de navegación, datos de contacto, sección hero, buscador rápido de propiedades, botones principales para buscar y publicar, sección de propiedades destacadas y tarjetas con foto, código, título, dirección, precio en pesos chilenos, valor en UF, dormitorios, baños y botón “Quiero saber más”.

Prompt 5:
Mejora listado-propiedades.html para mostrar las propiedades como tarjetas responsivas. Cada tarjeta debe incluir imagen, código de propiedad, nombre, dirección o sector, precio en pesos chilenos, valor en UF, características principales y botones para ver más información y abrir la ubicación. El diseño debe adaptarse a escritorio y móvil usando solo CSS.

Prompt 6:
Mejora buscador-propiedad.html para que tenga un formulario visualmente ordenado y accesible. Debe permitir buscar por operación, tipo de inmueble, provincia, comuna y sector. Agrega botones para buscar, buscar en mapa y buscar por código de propiedad. Todo debe funcionar como navegación HTML estática, sin JavaScript.

Prompt 7:
Mejora publicar-propiedad.html según los requerimientos del PDF. El formulario debe pedir tipo de propiedad, fecha de publicación, provincia, comuna, sector, fotografías de 1 a 10, dormitorios, baños, área total del terreno, área construida, precio en pesos, precio en UF, descripción y características adicionales como bodega, estacionamiento, logia, cocina amoblada, antejardín, patio trasero y piscina. Usa solo HTML y CSS.

Prompt 8:
Mejora registro-propietario.html. El formulario debe incluir RUT, nombre completo, fecha de nacimiento, correo electrónico, contraseña, sexo, teléfono móvil y número de propiedad según Bienes Raíces. Indica que la cuenta queda pendiente hasta revisión del administrador. El formulario debe ser responsive, accesible y visualmente ordenado.

Prompt 9:
Mejora registro-gestor.html. El formulario debe incluir RUT, nombre completo, fecha de nacimiento, correo electrónico, contraseña, sexo, teléfono móvil y certificado de antecedentes en PDF. Prompt 10:

La página de recuperación debe tener un campo de correo electrónico y botón para enviar enlace de recuperación. ------------------------------------------------------ Prompts backend, paneles, base de datos y AWS ----------------------------------------------------------------------------------------

Validación de contraseña robusta
“Implementa validación en JavaScript para que las contraseñas de login y registros cumplan mínimo 8 caracteres, una mayúscula, una minúscula y un carácter especial. Agrega mensajes visibles y claros para orientar al usuario antes de enviar el formulario.”

Validación de correo electrónico
“Implementa una función JavaScript para validar el correo electrónico considerando solo un símbolo @, mínimo 3 caracteres antes del @ y al menos un punto después del @. Aplica esta validación en login, registro de propietario y registro de gestor.”

Hash de contraseñas con PHP
“Actualiza los procesos de registro para que la contraseña no se almacene en texto plano. Usa password_hash con PASSWORD_BCRYPT al insertar propietarios, gestores y usuarios administradores en la base de datos.”

Verificación de contraseña en login
“Modifica el login PHP para consultar la base de datos por correo, recuperar el hash de contraseña y validar usando password_verify. El sistema debe responder con mensajes claros si el correo no existe, la contraseña no coincide o la cuenta no está activa.”

Sesiones PHP
“Al iniciar sesión correctamente, crea variables de sesión con session_start y $_SESSION para id, nombre, correo, rol y estado activo. Usa estas variables para mantener sesión en páginas protegidas.”

Protección de páginas privadas
“Crea archivos de verificación de sesión para restringir el acceso a páginas privadas. Si no existe sesión activa, redirige a error_sesion.html. Agrega verificaciones específicas para administrador, propietario y gestor.”

Cierre de sesión
“Crea la funcionalidad de cerrar sesión destruyendo la sesión con session_destroy y redirigiendo al login con un mensaje de salida exitosa.”

Panel administrador
“Mejora administracion.php para que funcione como panel real de administración. Debe listar propietarios registrados, permitir activar cuentas pendientes, eliminar cuentas, listar gestores postulantes, aprobar o rechazar gestores y mantener una vista clara de acciones administrativas.”

Separación de roles
“Corrige el flujo de acceso para que usuarios propietarios y gestores no puedan ingresar al panel administrador. El dashboard debe redirigir según rol: administrador a administracion.php, propietario a panel_propietario.php y gestor a panel_gestor.php.”

Panel propietario
“Crea o mejora el panel de propietario para mostrar el estado de cuenta, datos personales, número de propiedad, accesos rápidos, intereses, propiedades guardadas, búsquedas recientes y propiedades publicadas. Mantén la visual limpia y consistente con el sitio.”

Listado de usuarios validados
“En el panel administrador, agrega un registro completo de propietarios pendientes y activos. El administrador debe poder ver usuarios validados y eliminarlos posteriormente por limpieza, solicitud del usuario o falta de uso.”

Panel gestor
“Desarrolla un panel para gestores inmobiliarios aprobados. Debe mostrar propiedades asignadas, solicitudes de visita asignadas, estado de gestión y acciones para actualizar el avance de cada solicitud.”

Trazabilidad de visitas
“Implementa la trazabilidad de solicitudes de visita. Cuando un visitante solicita una visita, la solicitud debe guardarse en la base de datos, quedar visible para el administrador y poder ser asignada a un gestor inmobiliario.”

Asignación de gestor
“Desde el panel administrador, permite asignar una propiedad o solicitud de visita a un gestor aprobado. La asignación debe quedar guardada en la base de datos con id del gestor, estado de gestión y fecha de asignación.”

Actualización de estado por gestor
“Permite que el gestor actualice el estado de las solicitudes asignadas, por ejemplo pendiente, contactado, coordinada, cerrada o rechazada, para mantener trazabilidad del proceso.”

Solicitud de visita con confirmación
“Corrige el flujo de solicitar visita para que al enviar el formulario no redirija sin contexto. Debe mostrar un mensaje de éxito o error y registrar la solicitud en la base de datos.”
